What it does

Built as a Claude Code and Cowork plugin. Works from your terminal or desktop.

🔍

Verified role sourcing

Acts as a senior technical recruiter. Finds roles, verifies each on the company careers page, and ranks by fit score, sponsorship signal, and compensation.

📄

Tailored CVs, not templates

A/B/C variant system orders your experience differently per role type — AI/LLM, data platform, or backend APIs. Each role gets its own PDF built for that JD.

🌐

Multi-ATS form filling

DOM-based filling via Claude in Chrome — not pixel clicking. Handles hidden file inputs, React comboboxes, and cross-origin iframes on Greenhouse, Lever, Workable.

✉️

Per-role cover letters

Cover letter paragraphs live in the role config. Generated as a matching PDF or pasted as text where the platform requires it.

🔒

Profile data stays local

profile.json, roles/, tracker.json are all gitignored. Your personal data never touches the repo.

📊

Application pipeline tracker

Track every application from draft through offer. Add notes after calls. View your pipeline grouped by status.

Skills reference

All five slash commands — arguments, behaviour, and examples.

/source

Find verified, currently-open roles matching your profile. Returns 20+ ranked roles with fit scores, CV variant suggestions, sponsorship signals, and cover letter angles.

Usage

/source                      # all defaults
/source UK                    # UK roles
/source remote backend       # remote + backend filter
/source EU "data platform"    # quoted multi-word type

Arguments

ArgumentTypeDescription
[country]stringTarget country or region (UK, EU, US, remote, Canada…)
[role_type]stringRole focus (backend, data platform, AI infrastructure, LLM evaluation)

Profile inputs accepted

Reads profile.json automatically. Also accepts: LinkedIn PDF export, CV PDF/DOCX, or pasted resume text.

Evidence standard: Only verified open roles count. Fabricated URLs, expired postings, and unconfirmed sponsorship claims are flagged explicitly.
/new-role

Scaffold a new role JSON config by scraping the job description and proposing a tailored config with CV variant selection.

Usage

/new-role https://boards.greenhouse.io/company/jobs/123
/new-role  # interactive — Claude asks for the URL

What it produces

A roles/<role_id>.json pre-filled with company, title, ATS platform detected, recommended CV variant, cover letter angle, and a custom_answers scaffold.

JD reading checks: Some job posts hide instructions (e.g. "include the word FLAMINGO"). The skill flags these so you can add them to custom_answers.
/generate-cv

Build an ATS-optimised CV PDF and matching cover letter PDF. Uses reportlab Platypus — single-column, no tables or images, passes ATS parsers reliably.

Usage

/generate-cv monzo_backend_2026

# Or via CLI:
python src/generate_application.py --role monzo_backend_2026
python src/generate_application.py --all     # all roles
python src/generate_application.py --list    # list roles

CV variant system

VariantFocusBest for
AAI / LLM / EvaluationAI infrastructure, RLHF, evaluation roles
BData Platform / PipelinesData engineering, Airflow, ETL, analytics infra
CSenior Backend / APIsBackend, distributed systems, fintech, platforms
/apply

Open the job URL in Chrome, fill every required field, upload CV and cover letter PDFs, answer custom questions, then hand off for EEO fields and Submit. Requires Claude in Chrome.

Usage

/apply monzo_backend_2026

Fills: name, email, phone, location, LinkedIn, GitHub, work authorisation, salary expectations, pronouns, relocation availability, "how did you hear", and all custom_answers from the role config.

/track

View and update your application pipeline. Wraps src/tracker.py. All data stays in tracker.json (gitignored).

Usage

/track                                        # full pipeline
/track monzo_backend_2026                    # detail for one role
/track monzo_backend_2026 applied             # update status
/track monzo_backend_2026 "Screen Thursday 3pm"

ATS platform support

Tested platforms with documented DOM quirks.

Greenhouse (direct)

Standard job board URL.

  • File inputs are hidden — requires JS to expose before upload
  • React comboboxes: click toggle, type to filter, click option

Greenhouse (iframe)

Embedded on company page.

  • Extract embed src, navigate as top-level page
  • Same quirks as direct once loaded

Lever

Clean DOM, visible file inputs.

  • No cover letter file upload — paste as text

Workable

File inputs visible by default.

  • React inputs: left_click then type (form_input alone won't trigger state)

Ashby

Planned. PRs welcome.

SmartRecruiters

Planned. PRs welcome.

CAPTCHAs: The workflow stops before Submit where CAPTCHAs appear. Bypassing bot detection is not implemented and won't be.

Human-in-the-loop design

The boundary between agent and human is explicit and intentional.

✅ Claude does

  • Fills name, email, phone, location
  • Uploads CV and cover letter PDFs
  • Answers work authorisation questions
  • Fills salary expectations
  • Answers "how did you hear"
  • Pastes cover letter text (no upload)
  • Answers custom_answers fields
  • Reports exactly what was filled

🚫 Claude never does

  • Click Submit or irreversible buttons
  • Fill EEO / voluntary self-identification
  • Enter passwords or credentials
  • Solve or bypass CAPTCHAs
  • Accept terms or consent banners

FAQ

Does Claude submit my application? ▼
No. Claude fills all fields it can verify, uploads your documents, then stops. You review and click Submit. This is intentional — you own the final action.
Does it work without Claude in Chrome? ▼
/source, /new-role, /generate-cv, and /track work without it. Only /apply requires the Claude in Chrome extension.
Will my profile.json end up on GitHub? ▼
profile.json, roles/, tracker.json, and generated/ are all in .gitignore. They cannot be committed.
What if my ATS platform isn't supported? ▼
You can still use /generate-cv for tailored PDFs and /track to log it. Claude can attempt unsupported forms without pre-loaded quirk workarounds — it just won't have platform-specific playbooks. Open an issue to add support.
Does it handle CAPTCHAs? ▼
No. The workflow stops before Submit, where CAPTCHAs appear. Bypassing bot detection is not implemented by design.
Can I adapt this for my own profile privately? ▼
Yes — that's the primary use case. Clone the repo, fill in profile.json, and your data stays entirely local. The repo contains only the generic tooling.
