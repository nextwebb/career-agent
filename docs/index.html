What makes it different

Not a template engine. Not a job board. A Claude Code plugin that runs the entire application pipeline from inside your terminal. Your data never leaves your machine.

Agent Skills

Profile Bootstrap

Build profile.json from your CV or LinkedIn PDF. Extracts work history, infers three CV variants, and writes every bullet four ways so you're ready to generate tailored applications immediately.

$ /setup-profile # upload or paste your CV in the same message
info

Evidence standard: Claude never invents facts not in the source. Fields it can't populate are left blank and flagged in the report.

Market Intelligence

Find verified open roles matching your profile from company career pages. Filters by country and role type, ranked by fit score against your profile variants.

$ /source Germany backend
info

Crawls company career pages directly. Results include fit score, sponsorship signals, and relocation requirements.

Role Config Scaffold

Scrapes the job posting URL, detects the ATS platform, selects the right CV variant, and generates a pre-filled roles/<role_id>.json.

$ /new-role https://boards.greenhouse.io/stripe/jobs/5891234
info

Auto-detects Greenhouse, Lever, Workable from the URL. Prompts interactively for anything it can't infer.

PDF Generation

Builds ATS-optimised CV and cover letter PDFs using reportlab Platypus. Single-column, Helvetica, no tables, no images, clickable links — passes ATS parsers that choke on two-column layouts.

$ /generate-cv stripe_backend_2026
info

Output: generated/stripe_backend_2026_CV.pdf and generated/stripe_backend_2026_CoverLetter.pdf

ATS Form Filling

Opens the job URL, fills every required field from your profile and role config, uploads PDFs, answers custom questions, then hands off to you for EEO and Submit.

$ /apply stripe_backend_2026
verified_user

Claude never fills EEO/voluntary self-identification fields or clicks Submit. You review and confirm.

Pipeline Tracker

View your application pipeline, update statuses, add notes. Stored in tracker.json (gitignored — stays local).

$ /track stripe_backend_2026 "screen"
info

Statuses: draft, applied, screen, interview, offer, rejected, withdrawn. Run /track with no args to view the full pipeline.

Get started in one command

Checks Python 3, installs reportlab, registers the plugin marketplace, installs the plugin, and scaffolds profile.json. Requires Node 18+.

Option 1 — One command
$ npx @nextwebb/career-agent
Option 2 — Claude Code marketplace
$ claude plugin marketplace add nextwebb/career-agent
$ claude plugin install career-agent@career-agent
Then
$ /setup-profile # upload your CV or LinkedIn PDF
$ /source Netherlands backend
$ /new-role <url>
$ /generate-cv <role_id>
$ /apply <role_id>

Common questions

Is my personal data safe? add
Yes. profile.json, roles/, and tracker.json are all gitignored. Your PII never leaves your machine. career-agent has no backend, no accounts, and no telemetry.
Does it click Submit for me? add
No. Claude stops at the final review page. It also never fills EEO/voluntary self-identification fields (gender, race, veteran status, disability). This boundary is intentional — you retain full control of the irreversible action.
What do I need besides Claude Code? add
Python 3.10+ with reportlab for PDF generation (pip install reportlab). Node 18+ if you use the npx installer. For form filling via /apply, you also need the Claude in Chrome extension — all DOM-based ATS automation runs through it. CV generation alone does not require the extension.
Why does /setup-profile exist? Can't I just fill profile.json manually? add
You can, but the schema is substantial — three CV variants, six impact statements, and four bullet sets per job role. For a 10-role history that's 40+ bullet sets. /setup-profile generates all of it from your existing CV or LinkedIn PDF in one pass.
My ATS platform isn't listed. Can I add it? add
Yes. Open an issue first to avoid duplication, then add platform-specific quirks to the apply skill. Target platforms: Ashby, SmartRecruiters, Taleo, iCIMS, BambooHR. See CONTRIBUTING.md.
